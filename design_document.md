# 《代码回响：第404号档案》——沉浸式叙事解谜网页设计文档

## 1. 项目概述 (Project Overview)

**项目名称**：代码回响：第404号档案 (Echoes in Code: Archive 404)  
**类型**：ARG (侵入式虚拟现实游戏) / 叙事网页 / Web交互艺术  
**核心概念**：**"所有的代码都会说话，只要你懂得倾听。"**  
玩家扮演一名被学校聘请来维护旧官网的**“网络管理员”**。随着深入，玩家发现网站源代码中充满了奇怪的注释、被隐藏的图层和无法删除的“错误日志”，这一切都指向一名在两年前神秘失踪的天才学生——**林小雨**。

**设计目标**：利用前端技术（CSS3D, Canvas, Web Audio API）创造视觉奇观与心理悬疑，通过“网页代码”这一程序员熟悉的媒介讲述一个关于**反抗学术造假与追求数字永生**的故事。

---

## 2. 故事背景 (Narrative Background)

### **表层故事 (The Surface)**
“某某区第一中学”是一所历史悠久、升学率极高的名校。学校官网常年展示着辉煌的竞赛成绩和完美的师资力量。为了迎接即将到来的百年校庆，学校聘请你（玩家）对官网进行一次彻底的“翻新与维护”。

### **深层真相 (The Truth)**
两年前，高二学生**林小雨**是学校计算机社团的社长，也是这套官网系统的初版开发者。她在维护数据库时，意外发现了学校为了提升升学率，长期篡改部分低分学生的成绩档案，甚至通过“劝退”手段逼迫差生转学。
林小雨试图举报，但被学校发现并以“入侵教务系统”为由开除学籍，随后她在巨大的心理压力下失踪（传闻已自杀，实则隐姓埋名成为了一名白帽黑客）。
在离开前，林小雨将证据碎片化，加密隐藏在了她亲手编写的网页代码中，等待有一天能被人发现。

---

## 3. 角色设定 (Characters)

1.  **玩家 (Player)**：网络管理员 / 数字考古学家。好奇心强，技术过硬。
2.  **林小雨 (Lin Xiaoyu)**：
    *   **身份**：失踪的前代码开发者，“幽灵”。
    *   **表现形式**：不会直接出现。她的存在通过**代码注释**（如 `// help me`）、**隐藏的DIV**、**故障的图片**以及**控制台报错信息**来体现。
3.  **管理员 (Admin/System)**：
    *   **身份**：学校的自动化监控程序 / 现在的教务主任。
    *   **表现形式**：弹窗警告、自动删除玩家发现的线索文件、试图覆盖被还原的真相。

---

## 4. 网页设计与剧本流程 (Web Design & Narrative Flow)

整个体验分为**三个阶段**：表象（Order）、崩坏（Chaos）、真相（Truth）。

### **阶段一：表象（Order）—— 完美的假象**
*   **视觉风格**：极简主义，高亮白底，红色主色调（校徽色），大量使用微交互动画（Hover效果），给人一种“严谨、高端、官方”的感觉。
*   **关键页面**：
    1.  **首页 (index.html)**：
        *   **内容**：轮播图展示快乐的学生，校长寄语，高考喜报。
        *   **伏笔**：轮播图中有一张集体照，如果我们长时间盯着看，或者快速切换，会发现有一个学生的脸是模糊的（林小雨）。
    2.  **荣誉殿堂 (history.html)**：
        *   **内容**：历年竞赛金牌得主列表。
        *   **伏笔**：2024年的记录与其前后的年份格式有一点点不对齐（CSS `margin` 异常），暗示有人动过手脚。

### **阶段二：崩坏（Chaos）—— 系统的裂痕**
玩家在浏览中无意触发了“机关”（如点击了某个不起眼的像素点，或在控制台输入了特定指令）。

*   **视觉风格**：Glitch Art（故障艺术）。页面开始出现色块偏移、文字乱码、图片反色。
*   **关键事件**：
    1.  **控制台 (Console)**：
        *   玩家按下F12，发现控制台不仅没有报错，反而输出了一行绿色的字：`> System.connect(Lin_Xiaoyu)... connection established.`
    2.  **404页面 (404.html)**：
        *   **常规**：原本是“页面未找到”。
        *   **异变**：页面背景变成黑底绿字的代码雨。中间的“404”字样开始变形，变成“404 FOUND”（**找到了**）。
        *   **交互**：鼠标移动会擦除黑色的遮罩，露出底下一张被撕碎的成绩单（证据1）。

### **阶段三：真相（The Deep Web）—— 代码的记忆**
玩家破解了入口（如在404页面输入了林小雨的学号），进入了隐藏的**“第0号服务器”**。

*   **视觉风格**：赛博朋克 / 终端风格 (Terminal)。全黑背景，霓虹色线条，Three.js构建的3D数据可视化效果。
*   **关键页面**：
    1.  **真相数据库 (database.html)**：
        *   **内容**：不再是网页，而是一个交互式的3D文件系统。玩家可以点击浮动的文件夹，查看被删除的邮件、真实的成绩单、林小雨的日记。
        *   **高潮**：屏幕中央出现一个输入框，要求输入“解密密钥”。线索藏在之前的每个页面里（比如首页Banner图的文件名是密钥的一部分）。
    2.  **结局**：
        *   输入正确密钥后，播放一段林小雨留下的录音（Web Audio API可视化波形），讲述真相，所有的网页瞬间恢复正常，但首页的Banner图变成了一张林小雨微笑着比“V”的照片，标题变成“永远怀念我们的首席开发者”。

## 5. 详细页面设计与技术实现 (Page Detail & Tech Specs)

为了在比赛中脱颖而出，我们需要展示技术深度。

### **设计 A： 首页 (index.html)**
*   **剧情点**：看似正常，但隐藏着进入“里世界”的入口。
*   **技术亮点**：
    *   **视差滚动 (Parallax Scrolling)**：背景图与前景文字滚动速度不同，增加层次感。
    *   **隐写术 (Steganography)**：在Banner图片的`alt`属性或文件名中隐藏线索。
*   **交互逻辑**：
    *   用户点击“校长寄语”中的“诚实”二字 5 次。
    *   **效果**：屏幕颜色反转（`filter: invert(1)`），持续0.5秒，随伴随电流滋滋声（Audio）。

### **设计 B： 师资队伍 (shiziduiwu.html) -> 变异为“监控中心”**
*   **剧情点**：原本展示老师风采，变异后显示这些老师如何“监控”学生。
*   **技术亮点**：
    *   **CSS Grid 动态重排**：正常的网格布局突然打乱，老师的照片开始旋转、放大。
    *   **Canvas 粒子效果**：老师的照片散开成粒子，重组成一只“眼睛”的形状，注视着玩家。
*   **文案变化**：
    *   原：“张老师 —— 数学特级教师，关爱学生。”
    *   变：“张老师 —— 重点关注对象管理者，清除计划执行者。”

### **设计 C： 竞赛荣誉 (jingsai.html) -> 变异为“被抹除的名单”**
*   **剧情点**：揭露林小雨的奖项被他人冒名顶替。
*   **技术亮点**：
    *   **DOM 操作与文本替换**：利用 JS 逐字替换文本（Typing Effect）。
    *   **SVG 描边动画**：奖状的边框变成带刺的铁丝网动画。

### **设计 D： 核心解谜终端 (terminal.html)**
*   **剧情点**：玩家与林小雨留下的AI对话，拼凑证据。
*   **技术亮点**：
    *   **拟真终端**：模拟 Linux 终端操作（`ls`, `cat`, `sudo`）。
    *   **WebGL (Three.js)**：背景是一个旋转的、破碎的数据地球。

## 6. 为什么这个设计能拿奖？(Winning Strategy)

1.  **立意深刻**：跳出了普通的“信息展示”或“简单小游戏”的范畴，探讨了数据伦理、教育公平等社会话题，评委更有共鸣。
2.  **技术综合性强**：
    *   **基础扎实**：HTML5/CSS3 布局 immaculate。
    *   **前端进阶**：Canvas/SVG 动画。
    *   **交互创新**：Web Audio API、Gamepad API（可选）、DOM Hacking 模拟。
3.  **情感体验**：通过“正常 -> 恐怖 -> 感动”的情绪曲线，给评委留下深刻印象。
4.  **形式新颖**：ARG（替代现实游戏）在国内高校比赛中相对少见，容易造成“降维打击”。
